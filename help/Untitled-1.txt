						<br>

							<ul>
						<li> <b>Abaqus FEA Report Generator App</b> is an <b>automation</b> of the FEA report generation process 
						required for the structural qualification of the Bit Design.</li>
					
							<br>
						
						<li>This app aids in building the report at the click of a button and eliminates the earlier time consuming process of accessing simulation results through a remote access to the Linux server </li>
							<br><br>
							</ul>
						<h3><li>Invoke the App</h3></li>
						<ul>
						<li> To invoke this app open a terminal in your abaqus working directory and type <b>fearep</b> and press enter.

				<h1 id="LINK3">Running App</h1>
				<hr>

				<ol>

						<li>It’s best to have all of your files in a directory on your data drive 
							and that prior to opening the models in Abaqus the user must set the work directory 
							to the directory that you’re working from.	This directory will be hereafter referred 
							as <b>current directory</b> in the instructions. </li>
						<li><b>Open</b> Abaqus and load the CAE (*.cae) file first followed by relevant solution (*.odb) file. Make sure to uncheck the 
							<b>"Read only"</b> option in the dialog box.</li>
						<li>Run the following post processing macros from script generator,</li>
							<ol type="a">
								<li>"o01_printMesh.py" - This will create the <b>Mesh Image</b> in the current directory</li>
								<li>"o02_printConstraint.py" - This will create the <b>Constraint Image</b> in the current directory</li>
								<li>"o04_printStressMaxP.py" - 
									<ol><li>This will create the <b>Max. Principal Stress Image</b> in the current directory. </li>
										<li>It will also extract the max value of principal stress at the blade root required in the conclusion of the report. </li>
									
									
										<figure>
											<img src="help_images/maxPstress.png" >
											<figcaption>Fig.2 - Abaqus Status Bar</figcaption>
										</figure>
										<b>Note</b><br>
										 - Extracting Max. Principal stress value from the odb file can be 
										time consuming depending on the number of nodes. <br>
										- The bigger the model, the longer the time it takes for the script to determine the max stress.<br>
										
									</ol><br>
								<b><u>Things to remember</u></b> while using <b>"o04_printStressMaxP.py"</b> updated script from script generator code,
								<ol>
									<li>CAE Model File(*.cae) and solution file (*.odb) both <b>must</b> be open for the script to run successfully.</li>
									<li>In case of any errors, at first, please make sure the name of of solution file (*.odb) is the same as that of jobname specified in the <b>input.txt</b> file.</li>
								</ol>
									<br><br>
								</li>
							</ol>
						<li><b>Close</b> Abaqus after this is done.</li>
						<li>Please copy the relevant <i><b>utility_files</b></i> folder from ideas simulation in the current directory.</li>
						<li>Ensure all other images as mentioned in the previous section are available.</li>
						<li>Once all the required files are in place the app can be run by pressing the <b>SUBMIT</b> button.</li>
						
						<li>On clicking the <b>SUBMIT</b> button the following events will occur in a sequence, </li>
						<ol type="a"><li>At first all entries made in the GUI are backed up in a file called <b>"fea.cfg"</b> saved in the current directory. </li>
						<li>These entries are reloaded every time the app is opened in the current directory and the user need not retype the entries.</li>
						<li>After that the app will perform a <b>check</b> of the required files for generating the report.</li>
						<li>In case a file is missing the app will show a warning with list of missing files which are required for
							the app to run.	</li>
						<figure>
							<img src="help_images/MissingFilesWarning.png" >
							<figcaption>Fig.3 - Missing File Warning Message.</figcaption>
						</figure>
						<li>Images from Abaqus odb are automatically renamed by the <b>script generator code.</b> </li>
						<li>In case you have manually added any images in the folder please make sure the files have the same names 
							as displayed in this help document <a href="#LINK2">Required Files</a> and you are good to proceed.</li>	
						<li>Once all the files are confirmed to be in place the app will first write a report in a 
							latex format also created in the working directory.</li>
						<li>The latex file is then used to print the pdf copy of the report.</li>
						<li>In case there is a problem with printing the pdf report
							the app will show a warning about missing information (like blank slide) as below,	</li>
							<figure>
								<img src="help_images/warningMessage.png" >
								<figcaption>Fig.4 - Warning message due to problem with pdf generation</figcaption>
							</figure>	
							
						<li>On Successful completion of the App process user shall see a pop up message on screen as shown below,</li>
							<figure>
								<img src="help_images/Picture2.png" >
								<figcaption>Fig.5 - Successful Run Message.</figcaption>
							  </figure>	
							</ol>
						<li>The <b>RESET</b> button will load the form with preset default (sample) values as reference for the user as shown in Fig.1.</li>
						
				</ol>

				<h1 id="LINK4">Error Handling</h1>
				<hr>
				The Report generator app has been made robust enough to avoid any sort of errors. Inspite of this, if any 
					errors do occur then they may be due to following reasons. 
				<ol>
					<li>Mostly errors occur due to a missing data
					like image files or other required files  <i>input.txt</i> file for illustration or a bad entry (like newline chraracters) into the field.</li>
					<li>Inadequate user permissions to read/write/execute into the working directory</li>
					<li>Kindly report your errors to <a href="mailto:pgorade@slb.com">pgorade@slb.com</a></li>
					<li>Please attach the <b>logfile.txt</b> generated by the app in your current directory along with the email. </li>
					
				</ol>
